# Feature 자동화 스킬

사용자가 개발할 기능을 설명하면, API 명세 작성부터 이슈/브랜치 생성, 백엔드/프론트엔드 병렬 구현, 커밋까지 전체 플로우를 자동으로 처리합니다.

---

## 실행 순서

### 1단계: API 명세 작성

`api-spec` 스킬을 Skill 툴로 호출하여 사용자가 입력한 기능 설명을 기반으로 API 명세서를 작성하세요.

- 명세 작성이 완료되면 결과를 **`API_SPEC` 변수에 보관**하세요 (이후 에이전트 프롬프트에 그대로 전달)
- 명세서는 대화 컨텍스트 안에 표시하여 사용자가 확인할 수 있도록 하세요

### 2단계: GitHub 이슈 + 브랜치 생성

`github-issue-branch` 스킬을 Skill 툴로 호출하세요.

- 사용자가 입력한 기능 설명을 그대로 전달하세요
- 스킬 완료 후 **이슈 번호**와 **브랜치명**을 기록하세요 (이후 단계에서 사용)

### 3단계: 백엔드 + 프론트엔드 병렬 구현

**반드시 단일 메시지에서 두 Agent 툴 호출을 동시에 보내 병렬로 실행하세요.**

각 에이전트 프롬프트에는 아래 내용을 포함하세요:

#### backend-dev-agent 프롬프트 템플릿

```
다음 기능을 구현해주세요.

## 기능 설명
{사용자가 입력한 기능 설명}

## API 명세
{1단계에서 생성한 API_SPEC 전체 내용}

## 작업 브랜치
현재 브랜치: {2단계에서 생성한 브랜치명}
이슈 번호: #{2단계에서 생성한 이슈 번호}

## 완료 후 커밋 지침
구현이 완료되면 반드시 아래 순서로 커밋을 수행하세요:

1. 백엔드 구현 파일만 선택적으로 git add:
   예) git add backend/src/...

2. 커밋 메시지 형식:
   feat: {구현 내용 한 줄 요약} #{이슈 번호}

3. 커밋 실행:
   git commit -m "feat: {구현 내용} #{이슈 번호}"

4. **git push는 절대 하지 마세요.**

커밋이 완료되면 "백엔드 커밋 완료: {커밋 메시지}"를 출력하세요.
```

#### frontend-dev-agent 프롬프트 템플릿

```
다음 기능의 프론트엔드를 구현해주세요.

## 기능 설명
{사용자가 입력한 기능 설명}

## API 명세 (백엔드 연동 기준)
{1단계에서 생성한 API_SPEC 전체 내용}

## 작업 브랜치
현재 브랜치: {2단계에서 생성한 브랜치명}
이슈 번호: #{2단계에서 생성한 이슈 번호}

## 완료 후 커밋 지침
구현이 완료되면 반드시 아래 순서로 커밋을 수행하세요:

1. 프론트엔드 구현 파일만 선택적으로 git add:
   예) git add frontend/src/...

2. 커밋 메시지 형식:
   feat: {구현 내용 한 줄 요약} #{이슈 번호}

3. 커밋 실행:
   git commit -m "feat: {구현 내용} #{이슈 번호}"

4. **git push는 절대 하지 마세요.**

커밋이 완료되면 "프론트엔드 커밋 완료: {커밋 메시지}"를 출력하세요.
```

### 4단계: 완료 보고

두 에이전트가 모두 완료되면 아래 형식으로 요약하세요:

```
✅ Feature 플로우 완료

- API 명세: {엔드포인트 수}개 엔드포인트 작성
- 이슈: #{이슈 번호} {이슈 제목}
  → https://github.com/Teach-D/coin-battle/issues/{이슈 번호}
- 브랜치: `{브랜치명}`
- 백엔드: 커밋 완료 ({커밋 메시지})
- 프론트엔드: 커밋 완료 ({커밋 메시지})

다음 단계: 검토 후 /pr-merge-dev 로 PR 생성 및 머지
```

---

## 주의사항

- **1단계(API 명세)와 2단계(이슈/브랜치)는 순서대로 실행**하세요. 이슈 제목에 기능 내용이 반영되고, 브랜치명이 에이전트에게 전달되어야 합니다.
- **3단계는 반드시 병렬**로 실행하세요. 단일 메시지에서 두 Agent 툴 호출을 동시에 보내세요.
- 에이전트에게 **push 금지**를 명확히 전달하세요. commit까지만 수행합니다.
- 에이전트 프롬프트에 **API 명세 전문**을 포함하세요. 에이전트는 대화 컨텍스트를 공유하지 않습니다.
- git add 시 **각 에이전트가 담당 영역 파일만** 스테이징하도록 지시하세요 (백엔드 ↔ 프론트엔드 파일 충돌 방지).
