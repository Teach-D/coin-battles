# GitHub 이슈 + 브랜치 자동화 스킬

사용자가 작업 내용을 설명하면, GitHub 이슈를 생성하고 연결된 브랜치를 만들어 로컬에서 체크아웃까지 자동으로 수행합니다.

## 레포지토리 정보

- **owner**: Teach-D
- **repo**: coin-battle
- **base branch**: dev

## 실행 순서

### 1단계: 작업 내용 파악

사용자의 설명에서 아래를 파악하세요:
- 무엇을 구현/수정하는지 (핵심 기능)
- 왜 필요한지 (배경/목적)
- 어떤 작업 항목들이 포함되는지

설명이 모호하거나 범위가 불명확하면 한두 가지만 짧게 질문하세요. 충분하면 바로 진행하세요.

### 2단계: GitHub 이슈 생성

`mcp__github__issue_write` 도구를 사용해 이슈를 생성하세요.

**이슈 제목**: 간결한 한국어 (20자 이내). 동사로 끝내지 말고 명사형으로.
- 예: `업비트 WebSocket 시세 수신 구현`, `JWT 인증 미들웨어 추가`, `PVP 매칭 로직 개선`

**이슈 본문** 형식 (아래 템플릿 그대로 사용):

```
## 개요

[2~3문장으로 이 이슈가 왜 필요한지, 무엇을 달성하는지 설명]

## 작업 내용

- [ ] [구체적인 작업 항목 1]
- [ ] [구체적인 작업 항목 2]
- [ ] [구체적인 작업 항목 3]
(작업 규모에 따라 항목 수 조절, 최소 3개)

## 관련 브랜치

(브랜치 생성 후 업데이트 예정)
```

### 3단계: 브랜치 이름 결정

이슈 생성 응답에서 **이슈 번호**를 확인한 뒤 브랜치명을 만드세요.

**형식**: `feat/[작업내용-영문-kebab-case]-[이슈번호]`

브랜치명 규칙:
- 영문 소문자 + 하이픈만 사용
- 핵심 키워드 2~4개 (너무 길지 않게)
- 이슈 번호는 반드시 끝에 붙임

예시:
| 작업 | 브랜치명 |
|------|---------|
| 업비트 WebSocket 시세 수신 | `feat/upbit-websocket-price-12` |
| JWT 인증 미들웨어 | `feat/jwt-auth-middleware-7` |
| PVP 매칭 알고리즘 | `feat/pvp-matching-algorithm-23` |

### 4단계: GitHub 브랜치 생성

`mcp__github__create_branch` 도구로 브랜치를 생성하세요.
- **branch**: 위에서 결정한 브랜치명
- **from_branch**: `dev`

### 5단계: 이슈 본문 업데이트

`mcp__github__issue_write` 도구로 이슈 본문의 `## 관련 브랜치` 섹션을 업데이트하세요.

```
## 관련 브랜치

`feat/브랜치명`
```

전체 이슈 본문을 다시 작성할 때 기존 내용(개요, 작업 내용)을 그대로 유지하고 관련 브랜치 항목만 채우세요.

### 6단계: 로컬 브랜치 체크아웃

`Bash` 도구로 로컬에서 브랜치를 체크아웃하세요.

원격 브랜치가 이미 생성되었으므로:
```bash
git fetch origin && git checkout -b [브랜치명] origin/[브랜치명]
```

이미 로컬에 같은 이름 브랜치가 있다면:
```bash
git checkout [브랜치명]
```

## 완료 메시지 형식

모든 단계가 끝나면 아래 형식으로 요약하세요:

```
✅ 완료

- 이슈: #[번호] [이슈 제목]
  → https://github.com/Teach-D/coin-battle/issues/[번호]
- 브랜치: `[브랜치명]` (dev 기반)
- 로컬: [브랜치명] 브랜치로 체크아웃 완료
```

## 주의사항

- GitHub MCP 도구가 없거나 권한 오류가 나면 사용자에게 명확히 알리세요.
- 이슈 생성이 실패하면 브랜치 생성도 중단하고 오류를 보고하세요.
- 브랜치 생성 실패 시 이미 생성된 이슈 번호를 알려주고, 브랜치만 수동으로 만들 수 있게 안내하세요.
